# MIDTS Document Ownership

Last updated: 2026-07-15

MIDTS uses two document categories. This is intentional and avoids moving working vendor files into the client-facing Document Suite.

## Workspace Document Suite

The Workspace Document Suite owns client/commercial documents that MIDTS presents, approves, sends, or bills from.

Current and planned Document Suite controlled documents:

- Quote snapshots and approved quote PDFs
- Client-ready proposal documents when Proposal Builder is implemented
- Client-facing invoice documents when Invoice rendering is implemented

Rules:

- Document routes live under protected Workspace routes.
- Public `/documents/*` routes must not be added.
- Client/commercial documents use backend snapshots and controlled render data rather than URL-entered identifiers.
- Workspace controls approval/release actions.

## Vendor Safe Package

Vendor Safe Package output remains vendor working material stored in Drive/Google Docs by the backend.

This is a formal exception, not a competing client document system.

VSP documents are used to brief vendors and collect prices. They are not the source of truth for client quote, proposal, invoice, or project-commercial release documents.

Rules:

- VSP remains generated by Apps Script/Drive as backend transport and working-file creation.
- VSP links may be shown in Workspace queues as source evidence.
- Client uploaded files may be copied into the VSP folder for vendor context.
- VSP must not expose public document routes.
- VSP must not replace quote/proposal/invoice Document Suite snapshots.

## Ownership Boundary

| Document type | Owner | Storage/rendering | Audience |
| --- | --- | --- | --- |
| Vendor Safe Package | Backend transport | Drive/Google Docs working files | Vendors/internal |
| Quote | Workspace Document Suite | Protected Workspace render + Drive PDF | Client |
| Proposal | Workspace Document Suite, when implemented | Protected Workspace render + Drive/PDF | Client |
| Invoice | Workspace Document Suite, when client-facing rendering is implemented | Protected Workspace render + Drive/PDF | Client |

## Launch Position

Sprint 6 resolves the VSP/Document Suite conflict by documenting VSP as vendor working files only. No architecture redesign is required.
